## 素材 1：Anthropic 推出 Claude Corps 公益計畫

Anthropic 剛宣布一個很酷的計畫叫 Claude Corps：他們要找 1000 個志工，教他們把 Claude 用得出神入化，然後配對到美國各地的非營利組織，全職駐點一年幫忙推動公益任務。

不是送錢、也不是送模型，是送「會用 AI 的人」。這思路蠻有意思的——與其讓 NGO 自己摸索怎麼用 AI，不如直接派一個人的進去。

這大概是目前最實際的「AI for good」案例了。

## 素材 2：Anthropic 研究員：用自家 AI 寫程式，產出變 4 倍

Anthropic 內部做了一個調查，130 位研究員自己估，用了 Mythos（他們的內部開發工具）之後，產出中位數大概是原本的 4 倍。

這不是外部行銷說法，是內部研究團隊自己投票的結果。而且他們還寫了一篇文章叫「When AI builds itself」，探討 AI 開始參與自身研發的各種效應。

當 AI 開始幫忙寫下一代的 AI，那個加速曲線會很可怕。

## 素材 3：Anthropic vs OpenAI：IPO 前夕的 AI 軍備競賽

Reuters 今天報導，Anthropic 已經秘密提交 IPO 申請，和 OpenAI 正在搶誰先上市。兩邊都在賽跑，因為誰先掛牌，就等於先定義市場怎麼估值這類公司，也先確立誰是 AI 產業的公認龍頭。

Anthropic 目前的數據：超過 30 萬企業客戶，Claude Code 年化營收突破 25 億美元。從 2025 年 9 月的 5 億到 2026 年 2 月的 25 億，半年翻五倍。

AI 產業的商業化速度，真的不是開玩笑。

## 素材 4：NVIDIA 談 OpenClaw：自主 AI Agent 的未來

NVIDIA 官方部落格發了一篇蠻有份量的文章，討論 OpenClaw 這類「常駐型自主 AI agent」對企業的意義。

重點：大部分 AI agent 是你下指令、它做事、做完就停了。但 OpenClaw 這類是「有心跳的」——它定期自己醒來檢查任務清單、判斷什麼該做、做完繼續等下一輪。只在需要人類決策時才跳出來。

這種從「工具」到「數位同事」的轉變，NVIDIA 認為會是下一波組織變革的核心。

## 素材 6：AI 寫程式工具讓程式碼產量暴增 180%，但……

研究發現：用 AI coding agent 的開發者，程式碼產量增加 180%。但實際部署到生產環境的程式碼，成長幅度遠低於這個數字。

換句話說：AI 幫你寫更多，但你還是得花時間審、改、測。產量 ≠ 產出。

這個數據對正在評估導入 AI 輔助開發的團隊來說，是很好的提醒。

## 素材 8：AgentForge——28 個讓 AI Agent 寫出可靠程式碼的技能包

GitHub 上有人釋出了一個叫 AgentForge（SkillEngine）的開源專案：把寫程式的 AI agent 需要的 28 種技能打包成一套——從測試驅動開發、code review、到部署流程都有。

概念很簡單：與其每次叫 AI「幫我寫個功能」，不如先裝好這些技能，讓它像一個有 SOP 的工程師那樣做事。

開源社群現在的方向很清楚：不是做大 model，是做會用 tool 的 agent。

## 素材 9：OpenDream——讓 AI Agent 有「記憶」還會「做夢」

一個叫 OpenDream 的開源專案蠻有趣的：它給 AI agent 加上了本地記憶，還有一個「做夢（dreaming）」的自動化機制。

白話說就是：agent 在空檔的時候不像傳統工具直接休眠，而是會自動整理今天的互動、歸納重點、甚至生成新的想法。

這讓 agent 從「每次對話都是全新開始」進化成「它記得上次聊過什麼」。長期來看，這是 agent 從工具變夥伴的關鍵一步。

## 素材 10：OpenClaw 生態系爆發——macOS 版、輕量 VM 版接連推出

OpenClaw 這個自主 agent 框架最近生態系很熱鬧：

- HolaClaw：專為 macOS 打包的發行版，讓 Mac 用戶一鍵部署
- strangeClaw：跑在 Firecracker microVM 裡的輕量自託管版本，隔離性更好

從一個開源專案到開始出現各種 distro，這通常是生態系要起飛的前兆。就像 Linux 當年也是先有核心，再冒出 Ubuntu、Fedora 各種發行版。

OpenClaw 正在從「開發者玩具」走向「真正的基礎設施」。
